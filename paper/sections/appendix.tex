\section{Supplementary Roadmap}

The appendix supports the benchmark claims in the main paper rather than serving as a storage room for extra leaderboards. In the current benchmark-only manuscript, the supplement is organized around the following support blocks:
\begin{itemize}[leftmargin=5mm,itemsep=1mm,topsep=1mm]
    \item \textbf{construction audit and artifact plan:} source-corpus scale, admission logic, split policy, and what exactly will be released;
    \item \textbf{contamination-resistance boundary:} what DOI-disjointness, deduplication, and noleak reporting do and do not claim;
    \item \textbf{P3 intervention provenance:} how the 36-task subset was sampled from the 156-task full bank;
    \item \textbf{statistical validity:} headline intervals, intervention delta intervals, and near-tie interpretation for the four main probes;
    \item \textbf{calibration references:} degenerate, anchor-fixed, and chance-level baselines that contextualize the frontier scores;
    \item \textbf{full leaderboard tables:} complete per-probe reporting beyond the headline rows used in the main text;
    \item \textbf{split-resolved tables:} dev / test\_id / test\_ood views showing that the main claims are not single-split artifacts;
    \item \textbf{reasoning coverage and model style:} why explicit reasoning or thinking coverage helps some stages more than others;
    \item \textbf{stage-gap diagnostics:} how far each monolithic model remains from the stage-wise frontier;
    \item \textbf{selection-generation protocol notes:} the frozen cleanup policy and residual malformed-output behavior behind the isomorphic probe;
    \item \textbf{Tier~3 controlled-prompt audit:} full per-condition results, C4 per-model breakdown, and per-task pattern analysis supporting the main-text null-to-validity interpretation;
    \item \textbf{CMA-ES classical optimization baseline:} full P2 results under 6-query and 40-query budgets;
    \item \textbf{metric definitions:} formal formulas for every headline and diagnostic metric (Section~\ref{sec:metric_formulas});
    \item \textbf{prompt templates and task examples:} exact prompts, oracle specification, run configuration, and annotated task walkthroughs;
    \item \textbf{representative failure cases:} one audited P1--P4 example for over-proposal, over-edit, continuity trapping, and policy mismatch;
    \item \textbf{circuit audit cross-domain validation:} full per-probe tables, profile scores, and cross-domain pattern summary.
\end{itemize}

\section{Construction Audit and Artifact Availability}

The current VEH suite is not a loose aggregation of prompts. Its optimizer-facing stages are traceable to a concrete literature-extraction and admission pipeline. The upstream extraction audit covers 209 papers and 2090 field rows, with 151 papers carrying at least one issue and 77 carrying a high-risk issue. From this audit, 71 cantilever candidates are forwarded to oracle review, and the current P2 backbone admits 52 cleaned structure-review anchors. The current benchmark then packages these anchors into split-specific task banks whose manifests store source tables, input manifest hashes, seeds, and artifact SHA256 values.

The release package is designed around these manifests. We will release the exact task JSONL files and manifests for P1--P4, the oracle and evaluator code, the ta[REDACTED] scripts, the admission and split reports, and the per-model JSONL run logs used for the main and appendix tables. The intended artifact boundary is therefore the same as the paper boundary: each reported benchmark table should be traceable to a specific manifest-backed snapshot rather than to an undocumented local run.

Contamination resistance in the current release is deliberately scoped. It means DOI/title-hash deduplication in the upstream paper registry, DOI-disjoint source-anchor separation for P2--P4, manifest-level lineage tracking, and evaluation on post-fix noleak P1 runs only. It does \emph{not} mean that we claim web-scale near-duplicate search over model pretraining corpora or model-specific pretraining decontamination. P1 is intentionally outside the source-disjoint claim and should be read as a matched robustness stress stage rather than as a held-out generalization benchmark.

\section{P3 Intervention Provenance}

The P3 intervention in the main paper is intentionally not a rerun over the entire 156-task bank. It is a frozen protocol audit on a 36-task subset selected by stratified sampling over \texttt{split} $\times$ \texttt{source\_p2\_subtype}. The selection rule is: draw three tasks per observed stratum, then top up to 12 tasks per split from the remaining pool while preserving split balance. This keeps every observed family visible while preventing the intervention table from collapsing to a single split.
